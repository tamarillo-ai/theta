# history

## 目的

- `.codex/history/` は、実行結果、詰まりどころ、改善点を task 単位で残すフォルダです。
- 後続 task で同種の失敗を避けるため、事実と改善チェックを短く残します。

## 置き方

- 1 run につき 1 file とします。
- file 名は `YYYY-MM-DD-<task>-run.md` とします。
- 新規作成時は `TEMPLATE.md` を複製して使います。

## 書き方

- `概要` `最も引っかかった点` `実際に起きた問題` `学んだこと` `改善チェックリスト` を基本構成にします。
- `改善チェックリスト` は、次回の理想論ではなく、再発防止したい問題を `問題:` で始めて列挙します。
- 事実と判断材料を優先し、長い経緯説明は増やしません。
- task 固有名は必要な範囲で残しつつ、他 task に転用できる表現を優先します。
