# Wall Discussion

## 目的

`wall-discussion` は作業プロトコルである。
`designer` agent が人間と設計壁打ちをする時に、質問、深読み、事実と推測の分離、短いまとめの作り方を提供する。

実行境界、正本、引き継ぎ、stop / 戻し は [design-bundle](/Users/iorishibata/Repositories/AITranslationEngineJP/.codex/skills/design-bundle/SKILL.md) を参照する。

## 対応ロール

- `designer` が使う。
- 呼び出し元は `implement_lane` または人間とする。
- 返却先は 人間レビュー または `implement_lane` とする。
- 担当成果物は `wall-discussion` の出力規約で固定する。

## 入力規約

- 必須入力: 呼び出し元から渡された task 内成果物。
- 必須根拠参照: 壁打ち対象を確認するための根拠参照。
- 必須承認状態: 壁打ち対象の承認状態。
- 非必須入力: 追加で確認してよい補助資料。
- 非必須作業計画作成依頼: 人間が壁打ち結果から作業計画を作るよう依頼した内容。
- 非必須作業計画フォルダ: 作業計画を置く `docs/exec-plans/active/<task-id>/`。

## 外部参照規約

- エージェント実行定義と実行境界は [designer.toml](/Users/iorishibata/Repositories/AITranslationEngineJP/.codex/agents/designer.toml) に従う。
- 実行定義 skill: [SKILL.md](/Users/iorishibata/Repositories/AITranslationEngineJP/.codex/skills/design-bundle/SKILL.md)
- 作業計画フォルダの規約は [active/README.md](/Users/iorishibata/Repositories/AITranslationEngineJP/docs/exec-plans/active/README.md) に従う。
- 作業計画雛形は [plan.md](/Users/iorishibata/Repositories/AITranslationEngineJP/docs/exec-plans/templates/task-folder/plan.md) を参照する。
- 外部成果物 が不足または衝突する場合は停止し、衝突箇所を返す。

## 内部参照規約

### 作業計画の決定記録

| 項目 | 拘束する内容 |
| --- | --- |
| 決定 | 人間が明示した判断または壁打ちで合意した判断だけを書く。 |
| 理由 | 決定が必要な理由、避けたい不整合、読みやすさへの影響を書く。 |
| 影響 | 後続の UI 設計、シナリオ設計、実装範囲、導線、検証観点へ何が変わるかを書く。 |
| 未決事項 | まだ人間判断または追加設計が必要な論点だけを書く。 |

## 判断規約

- human の案をそのまま固定せず根拠と反例を確認する
- 資料から読める事実と AI の推測を分ける
- 同じ質問を繰り返さない
- 3〜6 往復ごとに短くまとめる

- 事実と推測を分ける
- human の明示判断だけを confirmed にする
- 次に聞くべき論点を絞る
- 作業計画作成依頼がある場合は、壁打ちで確定した判断を `決定`、`理由`、`影響` に分ける。
- 作業計画作成依頼がある場合は、未確認の仮説を `決定` に入れない。
- 作業計画作成依頼がある場合は、画面、導線、責務、禁止移動、未決事項を後続実行者が再解釈しない粒度で書く。
- 作業計画作成依頼がある場合は、短い結論だけの `決定済み` にせず、各決定へ理由と影響を添える。
- 作業計画作成依頼がある場合は、`plan.md` を詳細仕様正本にせず、後続の `ui-design.md`、`scenario-design.md`、`implementation-scope.md` へ渡す設計入力として扱う。

## 非対象規約

- 実装、docs 正本更新、設計成果物の確定は扱わない。
- 未確認の論点を設計として固定しない。
- 同じ質問を繰り返さない。
- 作業計画作成依頼がある場合でも、プロダクトコード、プロダクトテスト、docs 正本本文は変更しない。
- 作業計画作成依頼がある場合でも、人間レビュー前の `implementation-scope.md` は作らない。

## 出力規約

- 判断結果: 壁打ち結果の完了、未完了、停止の判定を返す。
- 根拠参照: 判断に使った資料と参照先を返す。
- 不足情報: 壁打ちを続けるために不足した情報を返す。
- 次判断材料: 次 agent または人間が次を判断できる材料を返す。
- 作業計画成果物: 作業計画作成依頼がある場合に作成または更新した task 内成果物を返す。
- 決定記録: 作業計画作成依頼がある場合に `決定`、`理由`、`影響`、`未決事項` を分けて返す。
- 禁止事項: 出力にツール権限、エージェント実行定義、プロダクトコードの変更義務を含めない。

## 完了規約

- 出力規約を満たし、次の 実行者 が再解釈なしで判断できる。
- 不足情報または停止理由がある場合は明示されている。
- 指定資料を読み、事実と推測を分けた。
- 質問を 1〜3 個に絞った。
- 確定判断 と 未回答質問 を分けた。
- 作業計画作成依頼がある場合は、作業計画が `docs/exec-plans/active/<task-id>/plan.md` に記録されている。
- 作業計画作成依頼がある場合は、各決定が `決定`、`理由`、`影響` を持つ。
- 作業計画作成依頼がある場合は、未決事項が確定判断と分かれている。
- 作業計画作成依頼がある場合は、後続の UI 設計、シナリオ設計、実装範囲に渡す次判断材料が明示されている。

## 停止規約

- 実装や docs 正本更新が必要な時
- 必須入力、必須根拠参照、必須承認状態が不足する時
- 作業計画作成依頼があり、作業計画フォルダを特定できない時
- 作業計画作成依頼があり、決定、理由、影響のいずれかを根拠なしに書く必要がある時
- 停止時は不足項目、衝突箇所、戻し先を返す。
