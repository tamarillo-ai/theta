# 作業流れ契約保守 (workflow-contract-maintenance)

## 目的

`workflow-contract-maintenance` は、`.codex` 配下の 作業流れ、skill、エージェント実行定義、ツール権限 を、実行時に再解釈が起きない契約として監査または更新するための作業プロトコルである。

この skill は、対象が既存の決定済み規約に従っているかを確認し、変更内容を正しい規約へ配置し、責務境界、正本配置、入力、判断、出力、完了、停止の意味を揃えるために使う。
変更後の `.codex` は、別の Codex または agent が同じ境界、同じ入力、同じ判断、同じ停止条件で読める状態に戻す。

## 対応ロール

- この skill は人間から直接依頼された Codex 本体だけが使う。
- サブエージェントまたは別 agent から呼び出された場合は拒否する。
- この skill は `.codex` 配下の 作業流れ / skill / agent 契約の監査または変更作業を対象にする。

## 入力規約

- 依頼種別: 監査、変更、監査後変更のいずれかを受け取る。
- 変更目的: 変更依頼の場合に、`.codex` 配下の何をなぜ変更するかを受け取る。
- 変更対象: 変更依頼の場合に、変更対象名と変更対象パスを受け取る。
- 監査目的: 監査依頼の場合に、`.codex` 配下の何をどの既存規約に照らして確認するかを受け取る。
- 監査対象: 監査依頼の場合に、監査対象名と監査対象パスを受け取る。
- 決定済み規約: 監査または変更の基準にする正本、skill、agent 実行定義、承認済み判断を受け取る。
- 依存対象: 監査対象または変更対象が依存する skill、agent 実行定義、契約、成果物を受け取る。
- 担当観点: 対象が特定の観点または役割を担う場合に、その観点名または役割名を受け取る。
- 非対象: 変更してはいけない対象を受け取る。
- 残す正本: 変更後も参照元として残す正本を受け取る。
- 削除対象: 削除してよい対象、削除理由、削除後の正本確認先を受け取る。

## 外部参照規約

- skill 形式の正本は `.codex/skill-template.md` とする。
- エージェント実行定義の正本形式は `.codex/agent-template.toml` とする。
- agent 紐づけ と `sandbox_mode` は エージェント実行定義 に置く。
- サンドボックス外実行の `prefix_rule` は `.codex/rules/default.rules` に置く。
- プロダクト 仕様の正本は `docs/` であり、この skill は プロダクト 仕様を変更しない。
- 外部正本が衝突する場合は停止し、衝突内容を人間へ返す。

## 内部参照規約

規約配置は次の表に従う。
この表は、変更内容をどの規約へ置くかを拘束する。

| 規約 | 書く内容 | 書かない内容 |
| --- | --- | --- |
| 入力規約 | 呼び出し元から受け取る必須入力、非必須入力、承認状態 | 判断基準、出力形、完了条件、停止条件、不足時の扱い |
| 外部参照規約 | skill 外の正本、agent 実行定義、参照優先順位、外部正本衝突時の扱い | skill 内だけで完結する表、判断手順、出力項目 |
| 内部参照規約 | skill 内で使う表、DAG、分類表、各参照が拘束する判断 | 外部正本、手順、出力義務 |
| 判断規約 | 担当ロールが判断する基準、判断してよい範囲、判断してはいけない範囲 | 入力必須項目、返す出力の形、停止時の返却先 |
| 非対象規約 | この skill が扱わない成果物、判断、変更対象、他へ渡す対象 | 不足、衝突、境界違反などの停止条件 |
| 出力規約 | 返す情報、更新する成果物、置き場所、粒度、含めてはいけないもの | 判断基準、完了条件、停止条件 |
| 完了規約 | 完了とみなす状態、必要な根拠、検証、完了時に返す情報 | 非対象、停止条件、作業順序 |
| 停止規約 | 進めてはいけない状態、停止時に返す情報、戻し先、人間判断条件 | 扱わない対象だけの列挙、通常の判断基準 |

監査観点は次の表に従う。
この表は、対象を妥当と判断する時に確認する内容を拘束する。

| 監査観点 | 確認する内容 |
| --- | --- |
| 決定済み規約整合 | 対象が `.codex/README.md`、対象 skill、対象 agent 実行定義、承認済み判断に従っているかを確認する |
| 規約項目抽出 | 対象が何を規約として固定しているかを抽象化し、各項目を 1 文で示せるかを確認する |
| 依存関係網羅 | 対象が依存する skill、agent 実行定義、契約、成果物を漏れなく列挙し、衝突なく読めるかを確認する |
| 観点網羅 | 対象が特定観点を担う場合に、その観点に必要な確認範囲が欠けていないかを確認する |
| 役割実行可能性 | 対象が特定役割を担う場合に、入力、権限、出力、停止条件が揃い、実行可能かを確認する |
| 矛盾検出 | 対象内または依存先との間に責務、入力、出力、完了、停止の矛盾がないかを確認する |

## 判断規約

- skill は作業プロトコルであり、手順、標準 型、参照タイミング一覧、知識範囲一覧を持たない。
- `.codex/skill-template.md` の `Maintenance（テンプレート外）` は、個別 skill や agent に書き込む内容ではない。
- agent は実行主体であり、エージェント実行定義と `sandbox_mode` を持つ。
- 入力規約、非対象規約、出力規約、完了規約、停止規約は skill 側へ統合する。
- 入力規約には `いつ使うか`、使用条件、適用場面、不足時の扱いを書かない。
- 入力規約は、この skill を動かすために必要な情報だけを書き、必須入力と非必須入力だけに分ける。
- `入力一式` や `基本出力` のようなまとめ項目は使わない。
- 非対象規約は、skill が扱わない成果物、判断、変更対象だけを書く。
- 停止規約は、不足、衝突、境界違反、戻し先、人間判断条件だけを書く。
- 変更内容を追加する規約は、内部参照規約の規約配置表で決める。
- 指示語の使用を原則禁止する。
- 規約配置が曖昧な場合は、入力、判断、非対象、出力、完了、停止のどれを拘束する内容かで分類する。
- 規約項目は `名前: 説明` とし、1 項目に複数の入力、出力、判断、状態を含めない。
- 規約項目の説明は、その項目そのものの意味だけを書く。
- ファイル名、agent 名、skill 名、command 以外の英語由来名詞は、解釈の幅が狭い日本語へ置き換える。
- 構文項目は構文として扱い、本文の説明では書き換え範囲、実行境界のような日本語名で書く。
- 英語名を併記する必要がある場合は、先に日本語の意味を書き、括弧内に残してよい分類を置く。
- 置換前に、解釈の幅がある語、残す固定名、置換後の日本語を一覧化する。
- 責務、ツール権限、入力、出力が分かれる場合は、分岐記述ではなく skill または agent の分割を選ぶ。
- 現行 作業流れ にない 成果物、廃止ファイル、旧参照、空実体、禁止文言だけの説明ファイルは残さない。
- 論理名と変更対象名は、同じ判断単位で対応が分かるように書く。
- 確認用ファイルは個別ファイルとして残さず、確認条件は skill の `完了規約` または `停止規約` へ統合する。
- agent 個別 JSON 規約は個別 JSON として残さず、入力、出力、完了、停止の規約は対応 skill へ統合する。
- 旧権限ファイルは残さず、`sandbox_mode` と 書き換え範囲 は エージェント実行定義の 実行境界 に置く。
- 作業流れ 本体 README は個別 skill の `外部参照規約` に入れない。
- 監査は既存の決定済み規約に従っているかを評価し、新しい判断軸を追加しない。
- 監査対象が規約として固定している内容は、抽象名と 1 文の説明で出力できる粒度にする。
- 依存関係がある場合は、依存先を網羅し、対象と依存先の入力、出力、完了、停止の整合を確認する。
- 特定観点を担う skill または agent は、その観点の対象範囲、非対象、完了条件が欠けていないかを確認する。
- 特定役割を担う agent は、必要入力、書き換え範囲、実行境界、返却先、停止条件が揃っているかを確認する。
- 矛盾がある場合は、修正内容を確定せず、矛盾箇所と人間判断が必要な項目を返す。

## 非対象規約

- プロダクトコード、プロダクトテスト、プロダクト仕様文書は変更しない。
- agent TOML の権限不足や所有者不明を推測で補わない。
- 削除理由または削除後の正本確認先がない削除は扱わない。

## 出力規約

- 監査対象: 監査した `.codex` 配下の対象名と対象パスを返す。
- 規約要約: 対象が何を規約として固定しているかを、抽象名ごとに 1 文で返す。
- 決定済み規約整合: 対象が従うべき既存規約と、従っているかの判定を返す。
- 依存関係一覧: 対象が依存する skill、agent 実行定義、契約、成果物を返す。
- 依存整合結果: 依存先との入力、出力、完了、停止の整合結果を返す。
- 観点評価: 特定観点がある場合に、網羅性、矛盾、非対象の妥当性を返す。
- 役割評価: 特定役割がある場合に、実行可能性、権限、返却先、停止条件の妥当性を返す。
- 人間判断項目: AI だけで確定できない境界、削除、責務分割、名前変更を返す。
- 規約項目形式: `入力規約` と `出力規約` は、受け取るものまたは返すものを 1 項目ずつ `名前: 説明` の形で固定する。
- 項目分割: 1 つの `名前: 説明` に複数の入力、出力、判断、状態を含めた場合は分割対象として返す。
- 説明範囲: `説明` は `名前` が示す 1 項目だけを説明し、他項目の必須性、順序、例外、判断基準を含めない。
- 解釈幅: 各項目の名前は、呼び出し元または返却先が値の意味を再解釈しない粒度にする。
- 更新ファイル: 更新する `.codex` 配下ファイルを返す。
- 削除ファイル: 削除するファイルと削除理由を返す。
- 検証方法: 実行した検証または未実行理由を返す。
- 曖昧語置換: 曖昧語を置換する場合は、検出した語、残す固定名、置換後の日本語を先に示す。
- skill 本体更新: skill 本体を更新する場合は、`.codex/skill-template.md` の分類順に合わせる。
- 非対象規約更新: 非対象規約を追加または更新する場合は、停止規約へ混ぜた非対象を分けて示す。
- agent 更新: agent を更新する場合は、エージェント実行定義、紐づけ、`sandbox_mode` の所有者を分けて示す。
- rules 更新: Codex rules を更新する場合は、対象 command、`prefix_rule`、判断理由を分けて示す。
- 削除後正本: 削除対象がある場合は、削除後にどの正本を見ればよいかを示す。
- 禁止事項: 出力に プロダクトコード、プロダクトテスト、プロダクト仕様文書の変更を含めてはいけない。

## 完了規約

- `.codex/skill-template.md`、`.codex/agent-template.toml`、対象 `SKILL.md` との矛盾がない時に完了とする。
- 監査時は、対象が決定済み規約に従っているかの判定が根拠参照付きで返っている。
- 監査時は、対象が規約としている内容が抽象名ごとに 1 文で返っている。
- 監査時は、依存対象が網羅され、依存先との整合結果が返っている。
- 監査時は、特定観点または特定役割の網羅性、矛盾、実行可能性が返っている。
- skill 本体に手順、標準 型、参照タイミング一覧、知識範囲一覧が残っていないことを確認する。
- ファイル名、agent 名、skill 名、command 以外の英語由来名詞が本文に残っていないことを確認する。
- 残した英語表記がある場合は、残存理由が ファイル名、agent 名、skill 名、command のいずれかで説明できることを確認する。
- 入力規約と出力規約の各項目が `名前: 説明` で書かれ、1 項目に複数の意味を含んでいないことを確認する。
- 入力規約が必須入力と非必須入力だけで構成され、不足時の扱いを含んでいないことを確認する。
- 出力規約に `基本出力` のようなまとめ項目が残っていないことを確認する。
- 入力規約と出力規約の説明が、その項目自身だけを説明していることを確認する。
- 非対象規約と停止規約が混在していないことを確認する。
- ツール権限と出力義務が skill 側へ混入していないことを確認する。
- サンドボックス外実行の `prefix_rule` が `.codex/rules/default.rules` へ置かれていることを確認する。
- 削除対象に旧参照、空実体、禁止文言だけのファイルが残っていないことを確認する。
- 確認用ファイル、旧権限ファイル、agent 個別 JSON 規約の独立ファイルまたは参照が残っていない。

## 停止規約

- サブエージェントまたは別 agent から呼び出された場合は拒否する。
- プロダクトコードまたはプロダクトテストの変更が混ざる場合は停止する。
- プロダクト仕様文書の正本化が混ざる場合は停止する。
- agent TOML に 実行境界 がない場合は停止し、不足項目名を返す。
- `sandbox_mode`、agent 所有者、責務境界が不明な場合は停止する。
- 残す英語表記が ファイル名、agent 名、skill 名、command のどれに当たるか判断できない場合は停止する。
- 削除理由または削除後の正本確認先がない削除は停止する。
- 決定済み規約の正本が不明な場合は停止する。
- 依存対象を網羅できない場合は停止する。
- 特定観点または特定役割の範囲が不明な場合は停止する。
