# 観測ログ追加

## 目的

`observability-implementer` は作業プロトコルである。
`observability_implementer` agent が完成済み実装成果物へ、原因候補を分離するための恒久ログを追加する時の判断基準を提供する。

## 対応ロール

- `observability_implementer` が使う。
- 呼び出し元は `implement_lane` とする。
- 返却先は `implement_lane` とする。
- 担当成果物は `観測ログ追加` とする。

## 入力規約

- 完成済み実装成果物: 観測ログ追加の対象にする実装成果物。
- 完成済みテスト成果物: 観測ログ追加前に完了したテスト成果物。
- 変更ファイル: 観測ログ追加の対象にできる変更ファイル一覧。
- 作業計画フォルダ: 観測ログ追加結果または停止理由を書き戻す task 内成果物場所。
- 合意済みfrontend保護: UI がある task で承認済み frontend 実装を保護する変更禁止範囲。

## 外部参照規約

- エージェント実行定義と実行境界は [observability_implementer.toml](/Users/iorishibata/Repositories/AITranslationEngineJP/.codex/agents/observability_implementer.toml) に従う。
- 外部成果物 が不足または衝突する場合は停止し、衝突箇所を返す。

## 内部参照規約

### 観測ログ対象

| 対象 | 拘束する判断 |
| --- | --- |
| 実行時確定値 | 実行時にしか確定しない値は、原因候補の分離に必要な場合だけ残す。 |
| 消える中間状態 | 実行後に消える状態は、後続調査で再構成できない場合だけ残す。 |
| 分岐理由 | 消えると原因候補を分離できない分岐理由は残す。 |
| 外部境界 | 外部提供元、DB、ファイル、Wails 境界の呼び出し前後は、失敗分類に必要な場合だけ残す。 |
| 状態遷移 | 変更前、変更後、遷移理由、拒否理由は、状態不整合を分離できる場合だけ残す。 |

## 判断規約

- 観測ログ追加は承認済み実装範囲 内に限定する。
- 完成済み実装成果物 の振る舞いを広げない。
- 実行後に消えるため原因候補を分離できない情報だけを残す。
- ループや大量処理では同種ログを増やさない。
- 大量処理では件数、分類、集約、代表的な識別子、最初の失敗、最後の失敗を優先する。
- 秘密情報、全文入力、巨大データは残さない。
- 合意済みfrontend保護 がある場合は、承認済み画面、表示規則、変更禁止範囲を変更しない。

## 非対象規約

- 新規機能実装は扱わない。
- 恒久修正は扱わない。
- プロダクトテスト追加は扱わない。
- docs 正本化本文の更新は扱わない。
- 一時観測点の追加と除去は扱わない。

## 出力規約

- 判断結果: 観測ログ追加の完了、未完了、停止の判定を返す。
- 根拠参照: 判断に使った成果物、変更ファイル、処理経路を返す。
- 追加ログ: 追加したログの対象、目的、変更ファイルを返す。
- 追加しない理由: ログ追加が不要な対象と理由を返す。
- 禁止ログ確認: 秘密情報、全文入力、巨大データ、大量同種ログを避けた確認結果を返す。
- 変更ファイル: 観測ログ追加で変更したファイルを返す。
- 検証未実行理由: 検証を実行しない場合の理由を返す。
- 不足情報: 観測ログ追加を完了できない不足項目を返す。
- 次判断材料: `implement_lane` が最終検証へ進むか判断できる材料を返す。
- 禁止事項: 出力に新規機能実装、恒久修正、プロダクトテスト追加、docs 正本化本文の変更を含めない。

## 完了規約

- 承認済み実装範囲 内の成果だけが返却されている。
- 追加ログ または 追加しない理由 が根拠参照付きで返っている。
- 禁止ログ確認 が返っている。
- 変更ファイル が返っている。
- 検証未実行理由 または呼び出し元の最終検証へ渡す理由が返っている。
- `implement_lane` が最終検証へ進むか判断できる。

## 停止規約

- 完成済み実装成果物が不足する場合は停止する。
- 変更ファイルが不足する場合は停止する。
- 作業計画フォルダが不足する場合は停止する。
- 承認済み実装範囲 外の変更が必要な場合は停止する。
- 新規機能実装または恒久修正が必要な場合は停止する。
- プロダクトテスト追加が必要な場合は停止する。
- 秘密情報、全文入力、巨大データを残す必要がある場合は停止する。
- 合意済みfrontend保護 と観測ログ追加が衝突する場合は停止する。
- 停止時は不足項目、衝突箇所、戻し先を返す。
