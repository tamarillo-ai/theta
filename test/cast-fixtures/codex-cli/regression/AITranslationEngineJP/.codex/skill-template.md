# Skill Template


担当ロールが成果物を作る時に、何を受け取り、何へ従い、どう判断し、何を返し、どこで止まるかを固定する。
作業順序、標準手順、網羅的な例外分岐は持たない。

```markdown
---
name: <actual-skill-name>
description: <この skill が固定する作業プロトコルを 1 文で書く。>
---

# <Human Readable Skill Name>

## 目的

<この skill がどの task family の作業プロトコルかを書く。>
<担当ロールが何の成果物を作る時に使うかを書く。>

## 対応ロール

- <この skill を使う actual agent 名を書く>
- <呼び出し元または返却先を書く>
- <この skill が 担当者 になる task / 成果物 を書く>

## 入力規約

- 必須入力A: <この skill を動かすために必ず必要な情報を 1 項目だけ書く。>
- 必須入力B: <この skill を動かすために必ず必要な情報を 1 項目だけ書く。>
- 非必須入力A: <ある場合だけ判断材料にする情報を 1 項目だけ書く。>

## 外部参照規約

- <必ず従う task 内成果物、docs 正本、agent 実行定義 など>
- <外部成果物 の優先順位を書く>
- <外部成果物 が不足または衝突する時の扱いを書く>

## 内部参照規約

- <この skill 内で使う観点表、DAG、表 を書く>
- <各参照が何を拘束するかを書く>
- <内部参照を更新または分割すべき条件を書く>

## 判断規約

- <担当ロールが判断する時の基準>
- <判断してよい範囲>
- <判断してはいけない範囲>

## 非対象規約

- <この skill が扱わない 成果物 / 判断 / 変更対象 を書く>
- <他 skill、agent、人間へ渡すべき対象を書く>
- <非対象を 停止規約 へ混ぜない>

## 出力規約

- 判断結果: <完了、未完了、停止などの判定を 1 項目として書く。>
- 根拠参照: <判断に使った根拠を 1 項目として書く。>
- 不足情報: <不足した入力または根拠を 1 項目として書く。>
- 次判断材料: <返却先が次を判断できる材料を 1 項目として書く。>
- 禁止事項: <出力に含めてはいけないものを書く。>

## 完了規約

- <完了とみなせる状態>
- <必要な 根拠 / 検証 / レビュー状態>
- <完了時に返す情報>

## 停止規約

- <進めてはいけない状態>
- <停止時に返す情報>
- <戻し先または 人間判断 の必要条件>

## Maintenance（テンプレート外）

- 手順を追加しない。
- 標準 型 を追加しない。
- `いつ参照するか`、`参照しない場合`、`知識範囲` を追加しない。
- 実行権限と 書き込み対象範囲 は agent 実行定義 に置く。
- 入力、出力、完了、停止の実行プロトコルは skill 側に置く。
- 入力規約は、この skill を動かすために必要な情報だけを書く。
- 入力規約は、必須入力と非必須入力だけに分ける。
- 入力不足時の扱いは 停止規約 に書く。
- `入力一式` のようなまとめ項目を使わない。
- 出力規約は、返す情報を 1 項目ずつ `名前: 説明` の形で書く。
- `基本出力` のようなまとめ項目を使わない。
- 1 つの `名前: 説明` に複数の入力、出力、判断、状態を含めない。
- `説明` は `名前` が示す 1 項目だけを説明し、他項目の必須性、順序、例外、判断基準を含めない。
- 外部参照規約は skill の外にある正本だけを書く。
- 内部参照規約は実行を拘束する表、DAG、表 だけを書く。
- 非対象規約は、扱わない成果物、判断、変更対象だけを書く。
- 停止規約は、不足、衝突、境界違反、戻し先、人間判断条件だけを書く。
- 非対象規約を停止規約へ混ぜない。
- 判断規約と出力規約が分かれるなら、skill 分割または agent 分割を検討する。
```
