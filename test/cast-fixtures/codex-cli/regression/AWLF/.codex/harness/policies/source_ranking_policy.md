# 来源排序策略

## 目标

确保研究质量优先，减少因过时或低质量来源导致的错误决策。

## 排序

1. 官方文档、官方博客、官方规范
2. 维护者文档、发布说明、仓库 README / ADR
3. 成熟开源实现与高质量源码
4. 中立技术分析与权威社区总结
5. 普通论坛、问答和个人博客

## 选择规则

- 涉及“最新”“当前”“今天”“最近”的问题时，必须检查发布日期或验证日期
- 涉及推荐与选型时，至少引用一个官方来源和一个成熟实现来源
- 对同一结论存在冲突来源时，优先信任更接近事实源的材料

## 记录规则

- 在 `source_log` 中记录标题、链接、来源类型、验证日期、使用理由
- 若来源被拒绝，也应记录拒绝理由，避免重复走回头路

## 禁止事项

- 未标注日期就使用时效性结论
- 只凭搜索结果标题下结论
- 过度依赖单一厂商宣传材料而不核对实现或规范
