# 设计先行策略

## 目标

确保实现建立在已审查、已批准的设计之上，而不是边写边决定。

## 强制规则

- 所有非只读任务必须先产出 `design_packet`
- `design_packet` 必须先展示给用户，得到批准后才可进入 `IMPLEMENT`
- 若实现过程中发现需求、边界或关键假设变化，必须回到 `DESIGN`

## design_packet 最小内容

- 问题定义与成功标准
- 范围内 / 范围外
- 本地事实与研究结论摘要
- 复用/新接口决策
- 主方案与备选方案
- 系统影响、失败模式、迁移、回退
- 验证计划
- 待用户确认点

## 审批规则

- 用户只批准设计，不等于自动批准后续扩展范围
- 设计被用户修改后，应重新生成最新版本设计摘要
- 没有明确批准时，状态必须停留在 `WAIT_USER_APPROVAL`

## 禁止事项

- 先改代码再补方案
- 用“这是小改动”绕过设计审批
- 在设计未冻结前承诺具体实现细节
