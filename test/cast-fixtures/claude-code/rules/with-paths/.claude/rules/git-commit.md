# Git 提交规则（强制）

## 一、提交时机

**【强制】** 只有用户明确请求时才创建提交（说"提交代码"、"commit"）

**【禁止】** 主动创建提交、询问"是否需要提交"、暗示或建议提交

---

## 二、提交信息格式

**【强制】** 格式：`<type>(<scope>): <subject>`

### type（类型）

| 值 | 说明 |
|---|---|
| `feat` | 新功能 |
| `fix` | 修复 Bug |
| `improve` | 功能改进（非新功能） |
| `opt` | 代码优化（逻辑简化、可读性提升） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `style` | 代码格式（不影响功能） |
| `test` | 测试相关 |
| `docs` | 文档 |
| `chore` | 构建/工具链 |

### subject（主题）

**【强制】**
- 中文描述，不超过 50 字符
- 祈使句（动词开头：增加、修复、更新）
- 不以句号结尾

✅ `feat(device): 增加设备批量导入`
❌ `feat(device): 增加了设备批量导入`
❌ `feat(device): 增加设备批量导入功能。`

---

## 三、提交前检查

**【强制】** 执行前必须：
1. `git status` - 查看状态
2. `git diff` - 查看改动内容
3. `git log -5 --oneline` - 了解项目风格
4. 在创建提交前，必须使用 code-reviewer 智能体进行代码审查


---

## 四、禁止签名

**【强制】** 不允许包含 `Co-Authored-By` 签名

---

## 五、修改历史提交

**【谨慎】** `git commit --amend`、`git rebase -i`、`git reset --soft` 操作前必须：
1. 告知用户影响
2. 确认已推送的提交是否受影响
3. 警告历史丢失风险
